# CLAUDE.md

このファイルはClaude Codeがこのプロジェクトで作業する際に必ず読み込む、
プロジェクトの背景・ルール・注意点をまとめたドキュメントです。

## プロジェクト概要

株式会社ジエンジサービス（G&G Service）のコーポレートサイト（gngs.co.jp）を
リニューアルするプロジェクトです。現行サイトの分析結果と移行方針をもとに、
新しいNext.jsプロジェクトとして再構築します。

- 現行サイト: https://www.gngs.co.jp/
- 現行CMS: 自社開発カスタムCMS「Gware」（PHP系・動的）
- 移行先: Next.js（App Router）

## 開発者について（重要）

**担当者はHTML / CSS / JavaScriptの基礎知識のみで、フレームワーク開発は初めてです。**

作業する上で必ず守ってほしいこと:

- TypeScriptは使わない（JavaScriptのみ）
- Tailwind CSSなどのユーティリティCSSフレームワークは使わない（CSS Modulesなど、
  通常のCSSプロパティで書ける方法を使う）
- 新しい概念（コンポーネント、App Router、データフェッチなど）を使うときは、
  ファイルの先頭やコードコメントで「これは何のためにあるか」を日本語で
  簡潔に説明する
- 難しい専門用語を使うときは、初出時に一言かみ砕いた説明を添える
- 一度に大量の変更をせず、担当者が動作を確認しながら進められるよう
  小さな単位で提案・実装する

## 現行サイトの構造（移行元）

現行サイトの主要ナビゲーションと構成:

- Home（トップ）

## 既知の不具合・教訓（新プロジェクトに絶対に持ち込まないこと）

1. **Gitマージコンフリクトマーカーの本番混入**
   現行サイトの `/service/imo` ページで、`<<<<<<< HEAD` / `=======` /
   `>>>>>>> [コミットハッシュ]` が実際の画面表示にそのまま残っていることが
   確認されている。`.github/workflows/check-conflict-markers.yml` が
   push/PRのたびにマーカーを検出し、見つかった場合はCIを失敗させる。
   混入したまま公開されないよう、このワークフローを弱めたり外したりしないこと。

2. **セキュリティ侵害（カジノ系SEOスパムの注入）**
   現行サイトは第三者によるコンテンツ改ざん（日本語カジノ・ギャンブル系
   キーワードの大量注入、`/app-*`, `/app-casino-*` ディレクトリ配下）を
   受けた形跡がある。静的サイト生成（Next.jsのSSG/ISR）を活用し、動的な
   サーバー処理を最小限にすることで攻撃対象領域を減らす方針とする。

3. **メタ情報の使い回し**
   現行サイトは全ページで同一のtitle/description/keywordsを使い回しており、
   SEO上の課題となっている。新サイトでは各ページ・各Newsの記事ごとに
   個別のメタ情報（title, description, OGP）を設定できる仕組みにすること。

4. **社名の誤記**
   現行サイトのヘッダー/フッターのリンクテキストに「株式会社ジンエイド
   サービス」という誤記が存在する。正式名称「株式会社ジエンジサービス」に
   統一すること。

## コーディング規約

- コンポーネントは1ファイル1責務を意識し、大きくなりすぎたら分割を提案する
- CSSはページ／コンポーネント単位でCSS Modules化し、グローバルCSSは
  リセットCSSや共通変数（色・余白など）程度にとどめる
- マージコンフリクトマーカーの検出はGitHub Actions
  （`.github/workflows/check-conflict-markers.yml`）がCI上で行う。
  ローカルでも念のためコミット前に目視確認する


## 進め方の原則

- 大きな設計判断（技術選定、URL設計、コンテンツの統廃合など）は必ず
  担当者に確認してから進める
- 実装が完了したら、担当者が「まず何を見て、どう動作確認すればよいか」を
  毎回簡潔に案内する
